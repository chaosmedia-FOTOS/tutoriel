Le but de ce tutoriel est de faire communiquer un programme écris en ActionScript à un autre programme écris en Python par le réseau local. Nous allons créer un serveur python à l’aide du module SocketServer et un client en ActionScript avec la classe Socket. Les deux programmes se communiquerons à l’aide de chaînes de caractères.

La version de Python utilisé sera Python 3, mais le code devrais fonctionner correctement sur Python 2 sans aucune modification. Le code ActionScript sera compilé en AIR 14.0. Le protocole utilisé sera le TCP.

Les sockets

Lorsque deux ordinateurs se communiquent par le réseau, ils se servent des sockets. Les sockets sont une interface IO comme un fichier où on peut y écrire des donnés ou les lires.

Pour faire communiquer 2 ordinateurs en utilisant le modèle client/serveur, le client doit d’abord créer un socket qui se connecte à l’adresse ip du serveur et au port que le serveur écoute. Une fois la communication établie, le client et le serveur peut écrire des donnés dans le buffer du socket et flusher le buffer pour le vider et les envoyer. Le destinataire peut alors lire les donnés de son propre buffer et l’interpréter et faire ce qui veux de ces donnés.

Le protocole TCP ne garantie pas que tous le buffer sera envoyé immédiatement lorsqu'on le flush, ou que les donnés n'arriveront pas à sa destination dans le désordre ou combiné. C'est pour cette raison que si on envoie beaucoup de donnés en succession, il faut séparer les chaînes avec un caractère séparateur et leur assigner un timestamp si l'ordre est important, et de faire un split pour séparer la longue chaine de caractère en plus petite qu'il étaient originalement lorqu'ils ont été envoyé. Par ailleur, flash ajoute un caractère null (\0xx) à la fin de la chaîne. Il est recommendé de faire un strip() sur la chaîne pour éviter tout effet indésirable.

Le serveur

Le module à utiliser lorsque l'on créer un serveur en Python est SocketServer. Pour créer un serveur TCP, il suffit d'instancier un objet TCPServer et d'appeller la méthode serve_forever(). Le serveur restera ouvert jusqu'à ce qu'on le ferme manuellement avec la méthode shutdown() ou que le programme se ferme. Le constructeur de TCPServer prend en paramètre le domaine (sur quel carte réseau le serveur écoute; '' laisse Python choisir), le port et la classe qui définie les actions que le serveur effectue lorsqu'il reçois des donnés.

#!/usr/bin/env python import SocketServer import logging #Log setting logging.basicConfig(format='%(asctime)s %(levelname)s: %(message)s', level=logging.INFO) #Global HOST = '' PORT = 8888 try: server = SocketServer.TCPServer((HOST, PORT), SocketServer.BaseRequestHandler) logging.info('Server starting') server.serve_forever() finally: logging.info('Server stopping') server.shutdown() logging.info('Server stopped')

Le code ci-dessus fonctionne. Un client peut se connecter, mais il est promptement déconnecté. La raison est que nous avons définie que la classe qui détermine les actions à faire lorsqu'un client se connecte est BaseRequestHandler, qui par défaut fait... rien! C'est pourquoi il faut hériter de cette classe et écraser ses fonctions handle(), setup() et finish(), pour effectuer des actions et rendre le serveur utile.

class FlashServer(SocketServer.BaseRequestHandler): def handle(self): #Code des actions à faire durant la connection pass def setup(self): #Code des actions à faire au début la connection pass def finish(self): #Code des actions à faire à la fin de la connection pass # # [...] # #Lorsque l'on créer le serveur, on utilise notre propre classe au lieu de BaseRequestHandler server = SocketServer.TCPServer((HOST, PORT), FlashServer)

Il est important de noter que dès que la méthode handle se termine, la connection avec le client est fermé. C'est pour cette raison qu'il y a généralement une boucle à l'intérieur pour garder la connection active.

Pour lire le socket, on se sert de la méthode recv() de la propriété request. avec en paramètre la taille du buffer en octets. Cette méthode bloque l'exécution du programme jusqu'à ce qu'il reçois des donnés. Similairement, pour envoyer des donnés, on se sert de sendall(). de l'objet request. sendAll() flush automatiquement le buffer, donc il n'est pas nécessaire de le faire manuellement. Ci-dessous, un serveur d'exemple qui attend que le client lui envoie une chaîne de caractère et lui répond en retournant la même chaîne de caractère en majuscule.

#!/usr/bin/env python import SocketServer class FlashServer(SocketServer.BaseRequestHandler): def handle(self): #On attend que le client nous envoie des donnés self.data = self.request.recv(1024).strip() #On envoie la même chaine au client, en majuscule self.request.sendall(self.data.upper()) server = SocketServer.TCPServer(('', 8888), FlashServer) server.serve_forever()

Le client

La classe utilisé pour créer un client en ActionScript est Socket, qui est dans le package flash.net.Socket et elle est très simple à utiliser. Une fois initialisé, on peut se connecter via la méthode connect(), qui prend le serveur et le port en paramètre. Une fois que la connection est réussi, l'objet Socket émet un évènement Event.CONNECT. Inversement, lorsqu'il se déconnecte, il émet un évènement Event.CLOSE. Lorsque le socket reçois des donnés, l'évènement ProgressEvent.SOCKET_DATA est émit.

Pour écrire des données dans le buffer, on se sert de la méthode writeUTFBytes() qui prend un String en paramètre. Dépendament du système d'exploitation, le buffer est flushé automatiquement ou pas. Il est donc important d'appeller manuellement la méthode flush() pour s'assurer que les donnés sont envoyé. Pour lire les donnés, la méthode readUTFBytes() est utilsé, avec la taille du buffer en paramètre.

package { import flash.display.Sprite; import flash.events.Event; import flash.events.ProgressEvent; import flash.net.Socket; public class Main extends Sprite{ private var _socket:Socket; public function Main():void { _socket = new Socket; _socket.addEventListener(Event.CLOSE, onClose); _socket.addEventListener(Event.CONNECT, onConnect); _socket.addEventListener(ProgressEvent.SOCKET_DATA, onReceive); _socket.connect("localhost", 8888); _socket.flush(); } private function onClose(e:Event):void { trace("Déconnecté du serveur"); } private function onConnect(e:Event):void { trace("Connecté au serveur"); var chaine = "Hello world"; trace("Envoyé: " + chaine); _socket.writeUTFBytes(chaine); } private function onReceive(e:ProgressEvent):void { trace("Reçus: " + _socket.readUTFBytes(_socket.bytesAvailable)); } } }

Voici ce qui devrait s'afficher dans la console de ce client si utilisé en conjonction avec le serveur Python donné plus tôt.

Connecté au serveur Envoyé: Hello world Reçus: HELLO WORLD Déconnecté du serveur
